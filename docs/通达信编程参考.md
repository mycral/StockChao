# 通达信公式编程参考

## 一、公式基础

### 1.1 基本语法结构

通达信公式由**输出语句**和**赋值语句**组成：

```
输出名称:表达式;           { 输出语句，显示为指标线 }
变量名:=表达式;            { 赋值语句，不直接显示 }
输出名称:变量名;           { 将变量输出为指标线 }
```

- `:` 表示输出（显示在图表上）
- `:=` 表示赋值（内部计算，不直接显示）
- `;` 语句结束符

**示例：**
```tdx
MA5:MA(CLOSE,5);          { 输出5日均线 }
MA10:=MA(CLOSE,10);       { 赋值给MA10，不直接显示 }
KDJ:MA(CLOSE,20);         { 同一输出名可多次赋值 }
```

### 1.2 基本数据

| 名称 | 说明 |
|------|------|
| OPEN / O | 开盘价 |
| HIGH / H | 最高价 |
| LOW / L | 最低价 |
| CLOSE / C | 收盘价 |
| VOL / V | 成交量（手） |
| AMOUNT | 成交额（元） |

### 1.3 操作符优先级

```
()  →  单目运算（NOT, REVERSE等）  →  * /  →  + -  →  > < >= <= = <> !=  →  AND  →  OR ||
```

## 七、注意事项

1. **未来函数**：ZIG、PEAK、TROUGH、REFX等函数会引用未来数据，可能导致信号漂移
2. **跨周期引用**：只能从短周期引用长周期，不能反向
3. **复权处理**：使用前复权数据时，某些计算结果可能与实际不符
4. **选股公式**：最后一行应为条件判断，返回1表示选中
5. **周期限制**：COST、WINNER等函数仅对日线有效
6. **NODRAW**：不想输出但需计算的指标可用NODRAW修饰符
7. **DRAWNULL**：想让某周期不画线时返回DRAWNULL
